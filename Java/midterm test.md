### 객체 upcasting, downcasting
자식 class > 부모 class = upcasting
부모 class > 자식 class = downcasting
> (동물 > 인간 = downcasting, int > double = upcasting)

#### 숫자 range 변경시 data loss 관련 내용
ex) int형으로 약 10억이라는 값이 저장되어 있는데, 이걸 short로 표현하라 하면 ±32,767 정도의 범위에서 표현할 수 없음. > overflow로 인한 data loss
ex) float형으로 π 값을 저장했음. 이걸 int로 바꾸면 걍 3이 되어 버림. > 소수점이 버려지며 data loss

### 산술, 대입, 증감, 논리, 비교 연산에 대해 난해한 연산
전위식을 이용하여 꼬아서 낼 수 있음.
```ex)
int x = 4; int y = 5; 일때
int z = x++ + ++y; 이면, z의 값은?
```
x++의 경우 x = 4를 사용 후 ++ 처리 (후위식)

++y의 경우 y = 5에서 ++ 처리 후 사용 (전위식)

따라서 z = 4 + 6 = 10.

### case문법 일부와 switch문
```
switch (condition) {
	case 1:
		line;
		break;
	case 2:
		line;
		break;
	case n:
		...
		break;
	default:
		line;
}
```
##### 그냥 이걸 기억하기.
1. switch 이후 조건문
2. case에는 int, char, String, enum 가능
> boolean, double 안됨.
3. break는 맨 마지막 빼고 다 넣어주기.
> 안해주면 case문 뒤에 주르륵 실행되면서 결과 꼬임
4. 조건식을 묶을 수 있음.
> 단순히 OR이라 생각하면 편함.

### class 선언 관련 생성자와 (무)인자 생성자 
기본 생성자는 컴파일러가 자동으로 생성함.
```
class A {
	A() { }
}
```
> 기본 생성자의 형태
하지만 사용자가 해당 클래스의 생성자를 하나라도 만들경우 컴파일러가 자동으로 생성하지 않음.

##### this = 자기 자신을 참조하거나, 같은 클래스의 다른 생성자를 호출
##### super = 부모 클래스의 생성자를 호출

> 부모 클래스에 기본 생성자가 없을 경우, 자식 클래스에서 기본 생성자를 호출할 경우 super()가 호출되지 못해 오류가 발생

### 객체 생성시의 호출, 실행 구조 검증
만약 자식 클래스가 생성될경우
super() 자동 호출로 부모 생성자가 우선 실행되고 그 다음 자식생성자가 실행되는 구조.

#### 메서드 오버로드 - 메서드를 여러번 정의하는 방법.
> 매개변수로 각각의 선언을 구분할 수 있어야 함.

#### 메서드 오버라이드 - 부모 클래스의 메서드를 자식 클래스가 재정의하는 것.
> 상속관계(extends)여야 함.
> 실제 객체 기준으로 호출되는 메서드임. (동적)

#### 객체 치환 - 부모 타입으로 자식 객체를 참조하는 것
> Upcasting이라고 생각하면 편함.

#### Garbage Collector(GC) - 더이상 쓰지 않는 객체를 자동으로 메모리에서 할당 해제하는 방식
> 쓰지 않는다 = 어디에서도 참조하지 않음.

### 객체 생성 - 캡슐화(정보 은닉)이 핵심 개념으로 작동.
관련 접근 지시자 4가지
#### public
> 걍 웬만해서는 다 ok

#### protected(시험 필수) 
> 같은 패키지면 ok, 다른 패키지일땐 상속받으면 ok.

#### default
> 다른 패키지면 다 안됨. 같은 패키지는 ok

#### private 
> 클래스 다르면 걍 다 안됨.
