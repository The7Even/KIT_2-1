Java - Camel Case (카멜 표기법)
Ex) AutoVendingMachine, getAge(), studentName
주로 단어단위로끊으며, 따로 언더바나 공백으로 구분하는것이 아닌 대문자로 작성함으로 구분한다. 첫글자는 소문자로 하는경우가 많음.
my name is john같은 경우도 myNameIsJohn 와 같이 작성하는거임.

자료형 boolean, char, byte, short, int, long, float, double을 사용함.
boolean (True or False) >> 참/거짓이라고 0,1이 저장되진 않는다. 그저 정말 적힌 그대로 True/False만 가능. 
char (2 byte, 문자)
byte (1 byte, -128 ~ 127)
short (2 byte, -32768 ~ 32767)
int (4 byte, ±2.1b), long (8 byte ±9.2e18)
float (4 byte, ±3.4e38), double (8 byte, ±1.7e308)
>> double형은 약 2^1024까지의 숫자가 저장되지만 이보다 더 큰 숫자는 BigInteger를 사용한다. 최대는 21억바이트 (약 1기가바이트)만큼 할당하여 ±1e6.5e9라는 엄청나게 광활한 범위의 연산이 가능하다.

상수 선언은 final을 활용한다. 일반적인 변수 선언 앞에 final만 붙이면 변수가 아닌 상수가 되는 마법

자료형을 결정할때 var를 사용하여 선언 + 초기화를 해주면 알아서 자료형 골라서 할당해준다.
